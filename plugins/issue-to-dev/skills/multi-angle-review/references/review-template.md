# Review Comment Template

Use this template when posting the multi-angle review as a GitHub comment.

```markdown
## 🔍 Multi-Angle Review

> Automated review for #{issue_number} — {issue_title}

---

### 📊 PM Review — Business Value

**Value Assessment**: [High / Medium / Low]

- **Business alignment**: [How this aligns with product strategy]
- **User demand**: [Evidence of user need — upvotes, related issues, customer feedback]
- **Impact**: [Expected impact on users/business]
- **Success metrics**: [How to measure if this succeeds]
- **Priority recommendation**: [P0–P3 with rationale]

---

### ⚙️ Tech Lead Review — Feasibility & Scope

**Feasibility**: [Straightforward / Moderate / Complex]

- **Architecture impact**: [How this affects the system architecture]
- **Scope estimate**: [S / M / L / XL with breakdown]
- **Dependencies**: [External dependencies or blockers]
- **Risks**: [Technical risks and mitigation strategies]
- **Suggested approach**: [High-level implementation plan]
- **Breaking changes**: [Any breaking changes or migration needs]

---

### 🎨 UX/Design Review — User Experience

**UX Impact**: [Major / Minor / None]

- **User flow impact**: [How this changes user workflows]
- **Consistency**: [Alignment with existing UI patterns]
- **Accessibility**: [A11y considerations]
- **Design needed**: [Yes — new mockups required / No — can use existing patterns]
- **Edge cases**: [Important edge cases and error states to consider]

---

### 📋 Conclusion

| Dimension | Result |
|-----------|--------|
| **Recommendation** | [✅ Approve / ⚠️ Needs Discussion / ❌ Reject] |
| **Priority** | [P0 / P1 / P2 / P3] |
| **Estimated Effort** | [S / M / L / XL] |
| **Design Required** | [Yes / No] |
| **Next Step** | [Proceed to Design / Proceed to Dev / Request More Info] |

**Key Risks**:
1. [Risk 1]
2. [Risk 2]
3. [Risk 3]

**Proposed Implementation**:
[Tech lead's suggested approach in detail — this becomes the basis for Approval Gate 1]

**Action Items**:
- [ ] [Action 1]
- [ ] [Action 2]

---
*🤖 Generated by issue-to-dev pipeline*
```
